# 机器人检测维保与二手交易平台 MVP PRD（V1）

## 1. 产品目标
- 在 8 周内上线可运营版本，打通 C 端与 B 端核心流程。
- 完成三条闭环：检测预约、维保工单、二手信息撮合。
- 形成可复用的数据资产：设备档案、检测报告、交易线索。

## 2. 角色定义
- C 端用户：设备拥有者、二手买家、二手卖家。
- B 端门店：店长、技师、客服。
- 平台运营：审核员、风控员、城市运营。

## 3. MVP 范围（必须上线）

### 3.1 C 端
- 注册登录（手机号+验证码）
- 设备档案创建与查看
- 发起检测/维修预约
- 查看检测报告与报价
- 发布二手信息与接收询价

### 3.2 B 端
- 门店入驻与资质提交
- 工单接单、检测录入、报价提交
- 订单状态流转（待检测/待确认/维修中/已完成）
- 二手线索响应（报价、备注、可服务时段）

### 3.3 平台
- 商家审核与上下架管理
- 交易信息审核（敏感词、虚假信息）
- 风险事件记录（异常设备、争议订单）
- 基础运营看板（工单、成交、时效）

## 4. 核心用户流程

### 4.1 检测流程
1. 用户添加设备并发起检测
2. 系统按城市+机型分配门店
3. 门店完成检测并提交报告
4. 用户确认报告并选择后续服务

### 4.2 维保流程
1. 用户基于检测报告发起维修
2. 门店提交报价与维修计划
3. 用户确认后进入维修
4. 完工后提交验收与评价

### 4.3 二手撮合流程
1. 用户发布二手信息（可附检测报告）
2. 买家发起询价，卖家响应
3. 双方线下/第三方完成交付
4. 回填成交结果与评价

## 5. 非功能要求
- 性能：核心页面 P95 < 2.5s。
- 可用性：SLA 99.5%（MVP阶段）。
- 审计：关键操作需保留日志 180 天。
- 安全：敏感信息加密存储、接口鉴权、频控。

## 6. 验收标准（8周）
- 检测预约成功率 >= 95%
- 工单完结率 >= 85%
- 二手发布到首次询价中位时长 <= 72h
- 用户满意度 >= 4.5/5
